# 0013 — 供應鏈題目三段管線 + 物件抽象判準(細節跟供應鏈意義)

- 狀態:Accepted
- 日期:2026-06-19
- 相關:[0001](0001-engine-content-separation.md)、[0002](0002-geometry-as-code.md);實作於 commands + skills

## Context

早期建供應鏈題目是「臨場」做的:邊查邊設計邊建,導致兩個問題——
1. **研究/準確性不一致**(公司代號靠記憶、流程細節隨手寫)。
2. **模型太粗糙**:每個環節一個方塊。後來釐清「抽象 ≠ 簡陋」,但需要一個明確判準,否則「要多細」沒準繩。

## Decision

### A. 三段管線(職責分離,各段產物可審查)
1. **研究 = 事實** — `/research-supply-chain <產業>` → `docs/supply-chains/<slug>.md`。WebSearch 查證實際運作、環節、流動、公司(代號查證)。
2. **設計 = 取捨** — `/design-demo <slug>` → `specs/<NN>-topic-<slug>.md`。精選節點、決定形狀/大小/布局/互動/物流。套 `object-abstraction` skill。
3. **建模 = 實作** — `/add-topic` 依設計建 content;公司填 `companies.csv`。
- 每段人類可審查、可調:研究錯改研究、設計不滿改設計,再建。

### B. 物件抽象判準:細節層次 = 供應鏈的結構層次
- **「抽象」只指 primitive、低多邊形的「風格」**,不是減少部位。
- **建不建一個部位,看它有沒有供應鏈意義**:鏈上有對應(材料/零件/製程/供應商)→ 建,**不論大小**;
  沒有供應鏈意義的造型細節(紋理/油漆/椅墊造型/通用組裝五金)→ 省。螺絲螺帽看角色,非一律省。
- 一句話:**模型拆到多細,跟著供應鏈拆到多細。** 編進 `object-abstraction` skill。

## Consequences

- ✅ 供應鏈題目產出一致、可審查、準確、教學對齊(模型上每塊都對得上一個鏈環節/公司)。
- ✅ 「研究/設計/建模」分離 → 各自可重做,不必整包重來。
- ✅ 抽象有準繩:不再單方塊,也不會為擬真亂加無意義細節。
- ⚠️ 多了兩個前置步驟(研究、設計)才建模;但換來品質與可維護,值得。對 trivial 小題目可略過直接 `/add-topic`。
- ⚠️ 研究的公司對應仍 AI 起草、需人工查證(代號可查證,歸類待校)。
