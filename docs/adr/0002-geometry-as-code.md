# 0002 — 幾何是程式碼:機械件程式生成 primitive,有機件借模型不細分

- 狀態:Accepted
- 日期:2026-06-18

## Context

3D 模型有多種來源:手刻 Blender、買/下載素材庫、AI 生模型(Meshy/Tripo)、程式生成。
本專案是學習用模擬、精度不要求、非商用,但有兩個硬限制:
1. 由 coding agent(vibe coding)主導開發——AI 看得懂改得動**文字**,改不動二進位網格。
2. 拆解動畫需要**命名好的獨立零件**;AI 生模型與素材庫多為單一網格,拆件要重切,反而最累。

## Decision

依形狀分流,以「拆件難度」與「AI 可迭代性」為首要準則:

- **機械/方正件**(機櫃、變壓器、管線、機房)→ 在 R3F 用程式生成 primitive(Box/Cylinder/Cone/Tube);
  拼不出的孔洞/切角用 `three-bvh-csg` 布林運算。**不建模、不存 GLB。**
- **精密規律件**(齒輪、螺紋)→ 程式 CAD(CadQuery/OpenSCAD,Python 用 `uv`)產 GLB,當 `kind:"model"`。
- **有機件**(飛機、機器人、無人機)→ 借外部低多邊形模型,**整隻當一個零件不細分**,
  繞過「單一網格難拆件」的硬傷;素材來源/授權先回報人類,agent 不自行下載。

## Consequences

- ✅ 落在 vibe coding 甜蜜區:幾何即程式碼,AI 能整段生成、用一句話調參(「散熱鰭加密一倍」)。
- ✅ 拆解動畫幾乎免費:程式生成的每塊天生獨立,只要位移座標。
- ✅ 零成本、零授權問題(機械件部分)。
- ✅ 參數化元件(`<ServerRack rows={6}/>`)可重生、可累積成 `kit/`(程式版 kitbash)。
- ⚠️ 美術上限受 primitive 風格限制——但這正好對應參考產品的低多邊形風,且非商用可接受。
- ⚠️ 有機件仍需人工挑模型/查授權,是 agent 幫不上的牆(見 CONTEXT §5)。
- 統一畫風靠共用材質 + `<Environment>` 打光,把不同來源視覺拉齊(屬 engine `materials.ts`)。
