## 0. 現況問題診斷(看實機截圖)

| # | 問題 | 截圖現象 | 為何錯 |
|---|---|---|---|
| P1 | **flow 是封閉跑馬燈** | 藍球繞「前緣 L→R + 後緣 R→L」整圈跑 | 供應鏈是**單向** L→R,後緣反向跑讓人以為晶圓會繞回去 |
| P2 | **flow 不穿過站** | 球在 z=±0.4 沿邊跑,從機台旁邊掠過 | 看不出「晶圓走進每一站被加工」,流動與節點脫鉤 |
| P3 | **設備/材料卡在主線上** | equip 排在 矽晶圓↔代工 中間、晶圓「穿過它」 | 研究 §4:設備/材料是**側向注入**代工(餵機台耗材),不是晶圓會流經的一站 |
| P4 | **晶圓沒變晶片** | 全程同一種藍球 | 封測後該變「成品晶片」,顏色/形狀無區別,失教學點 |
| P5 | **代工內部看不到** | 核心 foundry 是個實心箱,招牌的「機台陣列繞圈」被擋住 | 最重要的一站內部互動藏起來了(剛好可用新「透視」解) |
| P6 | **全開名稱會擠成一團** | 名牌互相重疊 | 節點間距偏密 + 子部位名牌太多 |

## 1. 教學目標 + 取捨

一眼看懂「**一片晶圓如何單向走過供應鏈、被設備餵養、最後變成晶片進伺服器**」。
- 節點層:研究 6 環節 → **主線保留 5 個**(設計→矽晶圓→晶圓代工→封測→下游),**設備/材料移成側站**(餵代工,不在主線)。修 P3。
- 省略:代工內部數百道製程不逐道建;用 flow 在代工處**繞一圈** + 透視看內部機台陣列 象徵(修 P5)。
- 抽象風格不變(primitive、低多邊形),但結構照供應鏈意義拆細(沿用 07 的元件組合)。

## 2. 布局 + 相機(修 P1/P3/P6)

- **主線橫排 z=0,L→R 五站**,間距拉到 ~2.0(比 07 寬,緩解名牌重疊 P6):
  `設計 x=-4 → 矽晶圓 x=-2 → 晶圓代工 x=0(核心,最大,置中)→ 封測 x=2 → 下游 x=4`。
- **設備/材料移到側站**:`x=0, z=-1.4`(代工正後方),用一條短側向 flow 餵進代工 → 視覺上「從旁注入」,不在晶圓主線上。修 P3。
- 核心強調:晶圓代工最大(箱體最高最寬)、略抬高;設計最小(知識密集、無實體)。
- 輸送帶只鋪**主線五站**(x −4.8→4.8);側站不在帶上。
- 相機:`position [2.2, 4.0, 9.2]`、`target [0, 1.0, 0]`(前上方廣角)。

## 4. 互動(修 P5/P6)

- **拆解**:沿用全域放射(`magnitude` only);核心 foundry 幅度最大。側站 equip 往後方散開,不要疊回主線。
- **透視(新)**:foundry 外殼 `enclosure:true` → 「透視」按鈕看穿,露出內部**機台陣列 + 微影機 + 鏡頭**(這正是研究強調、原本被擋住的「代工內部互動」)。可一併把 equip 主機箱也標 enclosure(看內部製程腔)。
- **選取/標籤**:五主站 + 一側站各一 annotation 節點;子部位走 `partOf` 繼承(沿用)。flow 不可選。

## 5. 物流 / flow(核心修法 P1/P2/P4)

引擎 flow 是**閉合迴圈、單一顏色**(ADR-0012)。在這限制內用三段 flow 表達單向 + 變形:

1. **晶圓流 wafer-flow**(accent 藍):上層可見段 **L→R 穿過 矽晶圓→代工→封測 的站心**(y≈0.9,真的穿過機台);**回程段壓到輸送帶下方**(y≈0.15)藏起來 → 視覺上只看到單向往右,修 P1+P2。代工處 path 多繞一個小圈表「數百道製程循環」。
2. **晶片流 chip-flow**(新材質 `chip` 綠):**封測→下游**,同樣上可見、下回程。與晶圓流不同色 → 「晶圓變晶片」一目了然,修 P4。
3. **耗材注入 supply-flow**(metal-light,少量 3~4 球、快):**equip 側站 → 代工**(z:−1.4→0),短促往前餵 → 表「設備/材料側向注入」,修 P3 的動線。

> path 設計重點:可見段務必**穿過站心**而非沿邊;回程段 y 壓到帶下被 conveyor 擋住(不必真的隱藏,遮住即可)。
