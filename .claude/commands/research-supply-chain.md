# Research Supply Chain
> 研究一條產業供應鏈的「實際運作」,產出一份結構化文件到 `docs/supply-chains/`,
> 作為之後建/更新 3D 供應鏈模型(content JSON + companies.csv + 未來 flow)的依據。
> 這是「先做功課再建模」:研究與建模分離,文件先行。

## Variables
industry: $1 — 產業/題目(如「半導體」「CoWoS 先進封裝」「電動車三電」「離岸風電」)。

## Instructions
- **實際查證**:用 WebSearch / WebFetch 查,**不要憑記憶**。優先可靠來源(產業報告、公司年報/法說、財經媒體、維基)。
- **聚焦「怎麼跑」**:材料/產品依序經過哪些環節、每環節實際做什麼、環節之間**流動什麼**、有無回饋循環。這比「列公司」重要。
- **公司以台股(TWSE/TPEx)為主**,附**查證過**的代號;國際大廠(NVIDIA/ASML…)用英文 ticker。代號錯誤零容忍——查證。
- 區分**事實 vs 推測**;不確定的標 `(待查證)`。
- 輸出到 `docs/supply-chains/<kebab-slug>.md`(slug 由 industry 決定;已存在則更新)。用下面 `Format`。
- 不寫進 content JSON 或 companies.csv——這支只產**研究文件**;建模是之後 `/add-topic` 的事。

## Format(產出文件的結構)
```md
# <產業> 供應鏈研究

> 研究文件,供建/更新 3D 供應鏈模型用。代號以台股為主、已盡量查證;不確定標 (待查證)。

## 1. 總覽
<這條供應鏈在做什麼、最終產品、為何重要(2~4 句)>

## 2. 環節(依流程順序)
### ① <環節名>
- 做什麼:
- 輸入 → 輸出:
- 關鍵設備/元件:
- 材料:
<每個環節一節,依實際流程順序>

## 3. 流動與互動
<環節之間流動「什麼」(矽晶圓/晶片/電力/資料…)、方向、回饋循環。這是 flow 動畫與拆解順序的依據。>

## 4. 關鍵公司(台股為主)
| 環節 | 公司 | 代號 | 角色 |
|---|---|---|---|
<每環節 1~N 家;國際大廠列英文 ticker;代號查證過>

## 5. 模型化建議(對應引擎 schema)
- **parts**:每環節 → 建議 primitive(box 機台 / cylinder 槽塔 / cone / tube 管 / flow 流動粒子)。
- **拆解節點**:每環節 = 一個 annotation 節點(掛 §4 公司)。
- **flow 路徑**:材料流動的 path 走向(從哪到哪)。
- **視覺/形狀參考**:像什麼(機台陣列、晶圓、輸送帶、槽罐…)。

## 6. 待查證 / 限制
<哪些對應或數字是推測、需人工再確認>

## Sources
<用過的來源,markdown 連結清單>
```

## Report
- 回傳產出檔路徑 + 一句話摘要(幾個環節、涵蓋幾家公司)。
- 提醒:可用 `/add-topic` 依 §5 建場景、把 §4 公司填進 `companies.csv`。
