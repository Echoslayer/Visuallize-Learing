# Research Supply Chain

> 研究一條產業供應鏈的「實際運作」,產出一份**深度**結構化文件到 `docs/research/supply-chains/`,
> 作為之後建/更新 3D 供應鏈模型(content JSON + companies.csv + flow)的依據。
> 「先做功課再建模」:研究與建模分離,文件先行。**輸出要禁得起人類查證、夠細到能直接設計 demo。**

## Variables

industry: $1 — 產業/題目(如「半導體」「CoWoS 先進封裝」「電動車三電」「離岸風電」「機器人」)。

## Instructions

### 1. 查證優先(不要憑記憶)
- 全程用 WebSearch / WebFetch,**每個關鍵事實至少一個來源**;重要數字/代號**交叉比對兩處**。
- 來源優先序:公司年報/法說/官網 > 證交所 / 公會 / 產業報告(工研院、資策會、TrendForce…) > 財經媒體(今周刊、鉅亨、商周) > 維基。
- **代號錯誤零容忍**:台股(TWSE/TPEx)附查證過的 4 碼代號;國際大廠用英文 ticker(NVDA/ASML…)。對不上就標 `(待查證)`,不要猜。

### 2. 深度要求(這是重點 — 不要一句話帶過)
每個**環節**至少寫到:做什麼(製程/技術細節)、輸入→輸出、關鍵設備/元件/材料、**技術門檻/護城河**、**台灣在此環節的地位**、**規模或關鍵數字**、**瓶頸/卡脖子點**。
能寫滿才算合格——薄薄四行就是這支命令過去的毛病,要避免。

### 3. 公司要具體(不只列名字)
每家公司要講清楚:**在這環節具體供應什麼產品**、地位(龍頭/二線/利基)、可得的話附**營收暴露或市占**、主要客戶/綁定關係。
**每個環節給多家**(別只填一家占位 — 見 [[supply-chain-multiple-companies]]);多對多(一公司可橫跨多環節)。

### 4. 拉高到產業視角
除了「環節怎麼跑」,補上:**產業地圖**(上中下游分層)、**市場規模/成長**、**競爭格局**(國際主導 vs 台廠角色)、**趨勢與風險**(技術路線、capex 循環、地緣政治、供需缺口)。

### 5. 區分事實 vs 推測
確定的直接寫;推論的標 `(推測)`;查不到的標 `(待查證)`。模型化建議屬設計取捨,歸到 §8,不要混進事實描述。

### 6. 輸出
- 寫到 `docs/research/supply-chains/<kebab-slug>.md`(slug 由 industry 決定;已存在則**更新並補深**,不是覆蓋成更短)。
- 嚴格照下面 `Format`。**不要**寫進 content JSON 或 companies.csv —— 這支只產研究文件;建模是之後 `/design-demo` → `/add-topic` 的事。

## Format(產出文件的結構)

```md
# <產業> 供應鏈研究

> 深度研究文件,供建/更新 3D 供應鏈模型用。代號以台股為主、已盡量查證(見 Sources);
> 事實/推測/待查證已分標。由 `/research-supply-chain <產業>` 產出。

## 0. TL;DR
<3~5 條最關鍵的 takeaway:這條鏈在做什麼、卡脖子在哪、台灣的位置、最該認識的幾家公司。>

## 1. 總覽
<這條供應鏈做什麼、最終產品、為何重要、台灣的定位(4~6 句,有血有肉,不要套話)。>

## 2. 產業地圖(上中下游)
<一段文字 + 分層:上游(材料/設備)→ 中游(製造/組裝)→ 下游(品牌/應用)。
讓人一眼看懂價值鏈往哪流、附加價值集中在哪段。>

## 3. 環節(依流程順序)
### ① <環節名>
- **定位**:<一句話 — 這環節在整條鏈的角色>
- **做什麼**:<製程/技術細節,2~4 句>
- **輸入 → 輸出**:
- **關鍵設備/元件/材料**:
- **技術門檻 / 護城河**:<為何難、誰進得來>
- **台灣地位**:<全球領先 / 跟隨 / 缺席;為什麼>
- **規模 / 關鍵數字**:<產值、市占、良率、道數… 標來源;不確定標 (待查證)>
- **瓶頸 / 卡脖子**:<供給受限、單一來源、地緣風險…>
<每環節一節,依實際流程順序;環節數視產業 4~8 個>

## 4. 流動與互動
<環節之間流動「什麼」(矽晶圓/晶片/電力/扭矩/資料…)、方向、回饋循環。
這是 flow 動畫路徑與拆解順序的依據 — 講清楚主流動 + 側向注入 + 內部循環。>

## 5. 關鍵公司(台股為主)
| 環節 | 公司 | 代號 | 具體供應 | 地位 | 備註(市占/客戶/暴露) |
|---|---|---|---|---|---|
<每環節多家;國際大廠列英文 ticker;代號查證過;備註能補就補,不知標 (待查證)>

## 6. 競爭格局(國際 vs 本土)
<每段價值鏈誰主導(國際大廠/台廠/中國…)、台廠扮演什麼角色(代工/利基/全球龍頭)、
有無被卡或反過來卡別人的點。>

## 7. 趨勢與風險
<技術路線演進、capex/景氣循環、地緣政治、供需缺口、ESG/法規… 影響這條鏈未來的因素。>

## 8. 模型化建議(對應引擎 schema)
- **整體布局**:<產線走向/陣列/堆疊;像什麼>
- **parts(每環節)**:<建議 primitive — box 機台 / cylinder 槽塔 / cone / tube 管 / model 有機件 / repeat 陣列>
- **拆解節點**:<每環節 = 一個 annotation 節點,掛 §5 公司>
- **flow 路徑**:<材料流動 path 走向(從哪到哪、哪裡繞圈)>
- **形狀/視覺參考**:<像什麼:機台陣列、晶圓、輸送帶、槽罐、葉片…>

## 9. 名詞表
<縮寫/術語 → 白話解釋(EUV、CMP、OSAT、ABF、PCS…),讓非專家讀得懂。>

## 10. 待查證 / 限制
<哪些對應、數字、歸類是推測或需人工再確認;研究的已知缺口。>

## Sources
<分類列出(綜述 / 各環節 / 公司),每條 markdown 連結 + 一句話用途。>
```

## Report

- 回傳產出檔路徑 + 摘要:幾個環節、涵蓋幾家公司、幾個來源。
- 點出最不確定、最需人類查證的 2~3 點。
- 提醒下一步:`/design-demo <slug>`(設計取捨)→ `/add-topic <slug>`(建模 + 把 §5 公司填進 `companies.csv`)。
